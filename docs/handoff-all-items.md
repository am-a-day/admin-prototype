# Handoff: Каталог → Все позиции (аудит меню)

Источник истины: прототип [src/features/storefront/catalog-workspace.tsx](../src/features/storefront/catalog-workspace.tsx) (компоненты `OverviewWorkspace`, `CatalogFiltersPanel`, `FilterPanelRow`, `TableHeaderRow`, `AuditDishRow`, `RowMeta`, `StatusBadge`, `AttributeBadge`, `AuditDot`, `AuditRowActionsMenu`). Дизайн-исток: Figma `NEW Админка`, фрейм `979:10759` (LITE · заполнено). Стек: React + TypeScript + Tailwind, Radix DropdownMenu, иконки lucide + @phosphor-icons/react.

## 1. Обзор

Таб «Все позиции» — **аудит меню**, не управление структурой. Задачи пользователя: увидеть общее состояние меню, найти позиции с дырами (описание/вес/КБЖУ/перевод/фото), увидеть исключения (стоп, архив, скрыта, расписание, режим отображения), сфокусироваться фильтрами и перейти к исправлению.

Архитектурные инварианты (не нарушать):

1. «Разделы» и «Все позиции» — два индекса в **один общий редактор позиции**. Клик по названию открывает тот же редактор; после закрытия — возврат в тот же аудит-срез.
2. В аудите **нет управления порядком**: никаких drag-handle; строки идут по алфавиту внутри раздела.
3. Правило колонок: **колонка = только то, где «пусто = плохо» почти для любой позиции**. Значение — если короткое (вес), индикатор-точка — если бинарно (описание, КБЖУ), счётчик — для перевода. Всё, где пустота легитимна (скидка, теги, стикеры, опции, рекомендации, видео) — НЕ колонка: живёт в цене, мета-строке, бейдже миниатюры и фильтрах.
4. Инверсия шума: **норма тиха, дыра заметна, но не алармова**. «Заполнено» — маленькая зелёная точка; «нет» — серый em-dash. Единственный тревожный акцент — оранжевый чип «На стопе».
5. Bulk-действия — общий с «Разделами» компонент, один словарь. Чекбоксы в текущем макете **визуальные** (без логики) — см. §12.

- Панель фильтров: `w-[228px]`, bg `#fbfbf9`, `border-r`, вертикальный флекс `gap-4`, `px-2 pt-4`, скроллится.
- Контент-зона: `flex-1 overflow-y-auto px-6 pt-[14px] pb-10`; ширину списка НЕ ограничиваем (`max-w` убран — таблица тянется по колонкам).
- Два sticky-слоя: шапка таблицы `top-0 z-10 bg-white` (высота 58px, включает поиск h-34 + py-3), заголовки групп `top-[58px] z-[9] bg-white`. ⚠️ Смещение 58 = высота `TableHeaderRow`; при изменении высоты поиска/паддингов пересчитать.

## 4. Панель фильтров (`CatalogFiltersPanel` / `FilterPanelRow`)

- Заголовок «Фильтры».
- **Scope «Раздел»** — триггер на `#f0f0ea` rounded-8 p-1.5: белая иконка-плитка 18px (эмодзи раздела или `List`), название, бейдж-каретка. Меню (Radix): «Все разделы» + разделы с эмодзи и счётчиком (позиции раздела в текущем срезе). Scope — независимое измерение, **комбинируется с любым срезом**; пересчитывает все счётчики панели.
- Группы (подписи 12/medium `#a6a09b`, `gap-9px`): **Позиции** (Все / Активные / В архиве) · **Доступность** (На стопе / Скоро будут / По расписанию) · **Заполненность** (Без описания / фото / граммовки / КБЖУ / перевода) · **Возможности** (Без рекомендаций / С рекомендациями / Со скидкой / Со стикерами / С тегами / С опциями).
- Строка фильтра: `p-1.5`, метка + бейдж-счётчик. Выбранная — белая, `border #e7e5e4`, тень `0 0 2px`, текст `#292524`; невыбранная — прозрачная, hover `#f0f0ea`, текст `#79716b`.
- В коде остались фильтры без строк в панели (`quick:with-video`, `status:hidden`) — доступны программно, в UI Figma-макета их нет.

## 5. Заголовок среза (`OverviewStatusBar`)

- «Все позиции» (label среза) 14/medium + подзаголовок 13/regular: человеческий счётчик, `· раздел «X»` при scope, `· с учётом поиска` при поиске, затем **проблема-ссылка** «N на стопе» (если N>0 и срез ≠ стоп; клик применяет фильтр). Стоп-часть в макете — inline-ссылка (не отдельная кнопка), унаследованный `hover:underline`.
- Сводки процентов заполненности НЕТ (убрана при переносе — макет её не содержит).

## 8. Строка позиции (`AuditDishRow`, h-62, `border-b #e5e7eb`)

`[✓18] [img36] [название + RowMeta] … [Описание●] [Вес] [КБЖУ●] [Перевод 2/3] [Цена] [⋮40]`

- **Чекбокс** 18px (визуальный).
- **Миниатюра** 36px rounded-6: эмодзи/фото на `#f5f5f4`; нет фото → `ImageOff` `#bc4a08` на `#faf0e6` + title «Нет фото». Видео → бейдж `Play` 3.5px в правом-нижнем углу (белый круг + ring).
- **Название** 13px `#292524`, truncate, клик → редактор.
- **`RowMeta`** — сегменты через вертикальные волоски `#e7e5e4` (h-11px), в фикс. порядке:
  1. **статусы** — `StatusBadge`, `gap-1`: «На стопе» (замок+оранжевый), «С расписанием»/«Скоро будет» (часы+синий), «В архиве»/«Скрыта» (slate), «Без кнопки»/«Без цены» (нейтр. `#f5f5f4`);
  2. **атрибуты** — `AttributeBadge` `#f5f5f4` с иконкой (мегафон=стикер, тег, огонь=острое), через точки `•`;
  3. **счётчики** — «2 опции · N рекомендаций» 12px `#595959`, через точки.
  Пустой сегмент не рендерится (нет плейсхолдеров). Правила переполнения — см. §11.
- **Ячейки аудита**: Описание — `AuditDot` (точка `#006045` / em-dash). Вес — значение 13px или em-dash. КБЖУ — `AuditDot` (точка / полый кружок «частично» / em-dash), тултип «на 100 г». Перевод — «2/3» 13px. Все с title-тултипами.
- **Цена** по центру, при скидке — ниже перечёркнутая старая 11px `#a6a09b`.
- **Kebab** `DotsThreeVertical` 18px; меню: Открыть · Редактировать · | · Показать/Скрыть · На стоп/Убрать · В архив(danger)/Восстановить · | · Задать скидку · Управлять рекомендациями.

## 9. Плоский режим

Активен только при непустом поиске (`flat = query.trim().length > 0`): группировка и sticky-заголовки групп отключаются, раздел уходит в `RowMeta` (`showSectionMeta`). Сортировки в UI нет — список всегда по алфавиту внутри группы. (Прежние «сортировка по цене/статусу» и кнопки «К разделу»/«Свернуть всё» удалены при переносе на Figma-макет.)

## 10. Пустые состояния

Dashed-карточка `#fafaf9`, заголовок 16/medium + текст 14 `#79716b` (тексты — `OVERVIEW_FILTER_META`, тон позитивный). Кнопка: меню реально пусто (срез «Все», без поиска/scope) → primary «Добавить позицию» (единственная точка создания на странице); активный поиск → «Очистить поиск»; иной пустой срез → «Показать все позиции» (сброс фильтра+поиска+scope).

## 11. Переполнение мета-строки (политика)

Оси взаимоисключающи by design (см. §12 модель доступности): статус ≤1 из {архив, скрыта}, доступность ≤1 из {стоп, скоро, расписание}, отображение ≤1. Правило доминанты: «стоп + расписание» → показываем «На стопе», расписание в тултип; архив подавляет доступность. Итог: **≤2 статус-чипа**. Атрибуты (теги/стикеры, не ограничены) — первые 2 + «+N» в тултип, порядок стикеры→теги. Вытеснение при тесноте: счётчики опций → атрибуты → статусы (статусы не скрываются). Перенос на 2-ю строку запрещён (ломает h-62 и ритм).

## 12. Контракт данных (замена мока)

`getDishAuditState()` — захардкоженный мок по `dish.id`, заменить на API. Поля на позицию: `photo?`, `hasVideo`, `descriptionFilled`, `weight: string|null`, `nutrition: filled|partial|none`, `translations:{done,total}`, `oldPrice?`, `guestAttributes[]{тип:label|tag|spicy}`, `optionsCount`, `recommendationsCount`, `section`.

⚠️ **Модель доступности (enum, не флаги).** По согласованию: `orderability: can_order | stopped | schedule` (взаимоисключающие радио) + `unavailable_presentation: default | hide | coming_soon` + производный `effective_state` → ровно один чип доступности. Правило: архив подавляет доступность. Это чинит и «стоп + скоро» одновременно, и семантику фильтра «Скрытые».

⚠️ **Состояние «не применимо»** per-поле (минимум — правилом по типу раздела: напитки/товары без КБЖУ и граммовки). N/A-ячейка — тихий em-dash, исключается из процентов, счётчиков фильтров и проблем-ссылок. Без этого у заведений с баром аудит навсегда «красный».

## 13. Доступность

Есть: `aria-expanded` на заголовках групп, `aria-label` на чекбоксах и kebab, focus-visible ring `#292524/10`, title-тултипы на ячейках. Довести: доступные тултипы вместо title («Описание: заполнено»), клавиатурная навигация по строкам (↑/↓, Enter), порядок фокуса панель → заголовок → шапка → строки.

## 14. Вне MVP (план v2)

1. Кликабельные дыры: клик по em-dash = исправление поля на месте (подмножество редактора).
2. Bulk: подключить логику к уже нарисованным чекбоксам (строки + заголовок группы) + общий с «Разделами» selection-toolbar.
3. Compact mode + клавиатурная навигация (АМ).
4. N/A-состояние с настройкой применимости полей per-тип.
5. «Аудит раздела» из kebab раздела в «Разделах» → сюда со scope; тихий маркер «требует внимания» в строках «Разделов».
6. Быстрое «Убрать со стопа» кнопкой в строке среза «На стопе».
7. Вернуть при потребности: сортировку (+ плоский режим по ней), «К разделу»/«Свернуть всё», сводку процентов, срезы «Скрытые»/«С видео» в панель.
